# Идеи и аналитика для Jammer

## Монетизация и подписка

### Что можно сделать платным

**Индивидуальные авторы (Creator Plan ~$3-5/мес):**
- Кастомная ссылка профиля (subdomain) — сейчас есть заглушка, можно ограничить бесплатный план
- Приоритет в каталоге игр (promoted-бейдж)
- Аналитика посещений игры (сколько загрузок, откуда)
- Расширенный лимит файлов игры (сейчас 100MB → платно 500MB)
- Убрать рекламный баннер с портфолио

**Организаторы джемов (Jam Host Plan ~$5-10/мес):**
- Неограниченное количество кастомных критериев
- Кастомный брендинг джема (цвета, шрифт, логотип)
- Экспорт результатов в CSV/Excel
- Приоритет в каталоге джемов
- Премиум-поддержка через email

**Команды (Team Plan ~$8-15/мес):**
- Приватная командная страница (скрытая от публики)
- Корпоративный профиль с подписками
- Командные джемы (только для участников команды)

### Как принимать оплату

**Рекомендую:** Stripe (самое популярное в мире, работает через посредника с карт РФ)  
- Но: официально не работает с РФ юр. лицами напрямую  
- Обходной путь: Lemon Squeezy или Paddle (они merchant of record, работают с РФ разработчиками)

**Альтернативы для РФ:**
- ЮKassa (бывший Яндекс.Касса) — карты РФ, СБП, ЮMoney
- Robokassa — аналогично
- CloudPayments — широкая интеграция

**Рекомендованная схема:**
1. Сначала запустить Lemon Squeezy / Paddle (меньше бюрократии, быстрый старт)
2. Параллельно подключить ЮKassa для аудитории РФ

**Техническая реализация:**
- Добавить `subscription_plan` в модель User (`:free`, `:creator`, `:host`, `:team`)
- Добавить `subscription_expires_at` 
- Webhook от платёжной системы обновляет `subscription_plan`
- FeatureFlag или метод `user.can?(:custom_link)` для проверки доступа

## Управление сессиями в настройках

**Текущая проблема:** Сессии накапливаются в таблице Sessions бесконечно. У активного пользователя за год накопится тысяча записей.

**Что уже есть:** В настройках отображаются активные сессии и кнопка "Выйти из других сессий".

**Что добавить:**
1. Автоматическая очистка — удалять сессии старше 30 дней (cron-задача в whenever)
2. Показывать сессию как "текущая" если session_id совпадает
3. Кнопка "Завершить эту сессию" рядом с каждой (не только всё разом)
4. Показывать число активных сессий на странице настроек

## Что ещё прикольного добавить в настройки

1. **Тема оформления** — Light/Dark режим (уже есть в модели, интерфейс закомментирован — раскомментировать)
2. **Язык** — переключатель прямо в настройках (сейчас в хедере)
3. **Публичная почта** — показывать или скрывать email на профиле
4. **Экспорт данных** — скачать свои данные (GDPR-дружественно)
5. **2FA** — TOTP-аутентификация (big feature)
6. **Статус "Не беспокоить"** — выключить уведомления на период
7. **Connected accounts** — список OAuth-провайдеров (GitHub, Google), возможность отвязать

## Редизайн профиля

**Проблема:** Пустой профиль выглядит как набор "Информации нет". 

**Решение (реализовано в этом PR):**
- Профиль скрывается от других при `profile_hidden = true`
- Баннер для администраторов при просмотре скрытого профиля

**Дополнительные идеи:**
- Если профиль пустой — показывать call-to-action ("Заполните профиль, чтобы другие узнали о вас")
- Показывать статистику: игр загружено, в джемах участвовал
- Добавить секцию "Ссылки" — GitHub, itch.io, Twitter и т.п.
- Добавить "закреплённую игру" на профиле
- Показывать количество друзей и подписчиков на карточке
